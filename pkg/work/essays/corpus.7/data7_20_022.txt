Interfaces kann man verwenden, ähnlich
wie abstrakte Classes, um Strukturen
von Objekttypen vorzugeben (zum Beispiel Collection)
alle classes, die ein Interface "implementieren"
müssen auch all seine Methoden implementieren
interface foo {}
class bar implements foo, bar2 {}
Vorteil gegenüber abstract classes: 
unbeschränkt viel implements 
möglich.
