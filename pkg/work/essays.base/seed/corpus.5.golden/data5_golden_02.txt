Content-independant metadata (Inhaltsunabhängige Metadaten): Diese Metadaten beinhalten Information, welche vom Inhalt der jeweiligen Daten nicht abhängig sind. Als Beispiele kann man etwa den Speicherort und das Änderungsdatum einer Datei nennen. Content-dependant metadata (Inhaltsabhängige Metadaten): Diese Art von Metadaten ist vom Inhalt der jeweiligen Daten abhängig. Beispiele: Größe eines Dokuments, Anzahl der Farben, Anzahl der Zeilen. Inhaltsabhängige Metadaten können noch weiter unterschieden werden. Man spricht von direct content-based metadata (Direkte inhaltsgestützte Metadaten), contentdescriptive metadata (Inhaltsbeschreibende Metadaten), domain-independent metadata (Bereichsunabhängige Metadaten) und domain-specific metadata (Bereichsabhängige Metadaten). Direct content-based metadata (Direkte inhaltsgestützte Metadaten): Diese Metadaten stützen sich direkt auf den Inhalt der Daten. Ein bekanntes Beispiel dafür ist ein Volltextindex, der auf einem Text eines Dokuments basiert. Content-descriptive metadata (Inhaltsbeschreibende Metadaten): Diese Art von Metadaten beschreibt den Inhalt von Daten ohne auf diese direkt zurückzugreifen. Dazu Bedarf es oft des menschlichen Wissens und Erkenntnisvermögens. Beispiel: Beschreibung des Inhalts eines Bildes. Domain-independent metadata (Bereichsunabhängige Metadaten): Diese Metadaten beinhalten Informationen, welche unabhängig von der Anwendung oder vom jeweiligen Bereich sind. Als Beispiel kann man etwa HTML/SGML document type definitions anführen. Domain-specific metadata (Bereichsabhängige Metadaten): Diese Art von Metadaten beschreibt den Inhalt von Daten, die von der Anwendung oder vom jeweiligen Bereich der Informationen abhängig sind. Beispiel: Relief und Besiedelung einer GIS Anwendung und die Daten der Volkszählung für das Gebiet.
